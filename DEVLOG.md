# 개발일지

## 2026-07-19 — MVP 구축, Supabase 전환, 자유 대화형 개편

### 1. MVP 첫 구축 (오전)

기획서(기후 우울증 예방 앱 "새싹") 기준으로 첫 버전 개발.

- **스택**: React + Vite + Tailwind CSS v3, Solar LLM(Upstage)을 프론트엔드에서 직접 호출, 데이터는 localStorage
- **화면**: 시작 화면 → 대화+미션 통합 화면(감정 유형/에너지 레벨 **선택형 버튼** → 미션 카드 → 완료 체크) → 기록/대시보드
- 미션 카테고리(생활 실천/가치 연결/마음 챙김/함께하기)는 세션마다 다른 시작점에서 균형 배분(`getCategoryPlan`)
- Playwright로 전체 플로우를 헤드리스 브라우저로 검증 (실제 Solar API 응답 확인)
- GitHub 저장소(`kwonqbeen-prog/climatemood`) 생성, 초기 커밋
- **배포는 보류**: 이 시점엔 API 키를 프론트엔드에서 직접 썼기 때문에, 배포하면 공개 JS 번들에 Upstage 키가 그대로 노출되는 문제가 있어 GitHub Pages 배포는 하지 않기로 함

### 2. Supabase 전환 (오후)

사용자가 다른 곳에서 준비한 업데이트 버전(zip)을 로컬 프로젝트에 덮어씀 — 아키텍처가 크게 바뀜.

- **인증 추가**: 이메일/비밀번호 + 카카오 로그인 (Supabase Auth) — 원래 "로그인 불필요" 방침에서 전환
- **데이터 저장**: localStorage → Supabase Postgres(`missions`, `sessions` 테이블, RLS 적용)
- **API 키 노출 해결**: Solar API 호출을 브라우저 직접 호출 → Supabase Edge Function(`solar-proxy`) 경유로 변경. 실제 Upstage 키는 이제 Edge Function 시크릿에만 존재
- Supabase 프로젝트 생성, `.env.local`에 URL/Publishable Key 설정, `schema.sql` 실행, `solar-proxy` 함수 배포(Supabase CLI가 이 개발 환경에서 로그인 안 돼서 **대시보드 Via Editor**로 배포), `UPSTAGE_API_KEY` 시크릿 등록
- curl과 직접 API 테스트로 Supabase 연결·이메일 회원가입·Edge Function 호출까지 전부 확인

### 3. 카카오 로그인 트러블슈팅

가장 오래 걸린 부분. 순서대로 겪은 문제:

1. **KOE205** (동의 안 한 항목 요청) — 카카오 로그인 동의항목에서 "프로필 이미지", "카카오계정(이메일)" 활성화 필요
2. 이메일 동의항목이 **"권한 없음"**으로 아예 안 켜짐 — 개인 개발자 앱 기본 제약. Supabase의 "Allow users without an email" 토글은 이 문제를 해결하지 못함(요청 자체는 그대로 나감)
3. **해결**: 카카오 "비즈 앱" 전환(개인 자격, 사업자등록번호 불필요, 즉시 처리) → 이메일 동의항목 활성화 가능해짐
4. 비즈 전환 과정에서 Redirect URI가 초기화되어 **KOE006** 발생 → 재등록으로 해결
5. Client Secret 발급 위치가 콘솔 개편으로 "앱 설정 > 앱 > 플랫폼 키"로 이동해 있었음
6. 최종적으로 카카오 로그인 리다이렉트까지 정상 확인

### 4. 자유 대화형 개편 (저녁)

버튼 선택형(감정 유형/에너지 레벨) 대화를 자유 텍스트 입력 중심 구조로 전면 개편.

- **모드 분리**: `mission`(감정+에너지 파악 후 미션 제시) / `freechat`(자유·마음챙김 대화, 종료조건 없음)
- **통합 JSON 스키마**: 매 턴 `{mode, stage, reply_message, emotion_type, energy_level, suggested_chips, suggest_mission, mission_hint, missions}` 형태로 응답, 추천 칩(2~3개)을 함께 받아 렌더링
- 당일 첫 방문 시 엔트리 카드("오늘의 맞춤 미션 받기"), freechat 중 AI 판단에 따른 하루 1회 자동 미션 제안
- **실측 중 발견한 버그 2건과 수정**:
  - mission 모드 중에도 `suggest_mission:true`가 나와 카드가 중복 노출될 뻔함 → `mode==='freechat'`일 때만 인정하도록 프론트에서 필터링
  - "무기력해요 + 미션 3개"처럼 감정·에너지가 한 문장에 섞이면 종종 오추출 → 프롬프트에 한국어 단서→enum 매핑표 추가로 해결
  - 미션 생성 트리거를 모델의 `stage` 필드에만 의존하지 않고, 프론트가 "감정+에너지 둘 다 확정" 시점을 직접 감지하도록 변경 (모델이 stage 전환을 놓치는 경우가 있었음)
- 로그인 없이 Vite dev 서버 컨텍스트에서 실제 소스 모듈을 직접 호출하는 방식으로 대화 스키마를 검증 (이메일 confirm 게이트 때문에 헤드리스 로그인이 막혀서 찾은 우회 검증 방법)

### 5. 닉네임 입력 방식 변경 (밤)

## 2026-07-21 — "지구 마음" 리브랜딩 (구조 우선 패스)

사용자가 작성한 브랜딩/UI-UX 기획서에 따라 "새싹" → "지구 마음"으로 리브랜딩. 기획서에
정확한 컬러 hex 값과 카테고리별 색상 매핑이 "미확정"으로 명시돼 있어, 이번 작업은
**구조만 먼저** 진행하고 색상 확정은 후속 세션으로 미뤘다.

- **리브랜딩 텍스트/아이콘**: 로고 랙업 4곳(AuthScreen/StartScreen/NicknameScreen/
  ThemeOnboardingScreen), 페이지 타이틀, AI 페르소나 문자열, 채팅 헤더까지 "새싹"→
  "지구 마음" 전면 교체. 로고 아이콘도 `eco`(잎)→`public`(지구본)으로 교체. 브랜드
  카피 "마음을 돌보다, 지구를 돌보다"를 AuthScreen 부제와 StartScreen 태그라인에 배치.
  배포 인프라 식별자(package.json name, vite base path, GitHub 저장소명, Supabase 키,
  localStorage 키)는 기존 사용자 데이터·배포 URL을 깨지 않기 위해 의도적으로 유지
- **2계층 컬러 토큰 아키텍처**: 기능 레이어(중립 저채도 accent)와 감성 레이어
  (`--gradient-cta` 시그니처 CTA, `--gradient-planet-*` 마음 지구 상태별 그라데이션)로
  분리. 모든 신규 hex는 PLACEHOLDER로 명시(`src/index.css` 상단 체크리스트 참고).
  시그니처 CTA는 로그인/온보딩/대시보드 핵심 진입점 5곳에만 적용, 나머지 버튼은 기존
  `accent` 유지
- **"마음 지구"(MindPlanet) 컴포넌트 신규 추가**: 누적 완료 미션 수로 seed/growing/

  카테고리 무관하게 명도 3단계로만 변화(기획서 §6.2 준수). 애니메이션·위성/링 장식은
  기획서 자체가 후순위로 명시해 이번 범위에서 제외
- **기록 탭(DashboardScreen) 재구성**: 헤더 → 마음 지구 → 대시보드 박스(오늘 미션/
  성공 미션 통계 + "미션 구경하기" CTA 버튼) → 기존 재도전 섹션 → 기존 완료 히스토리
  순으로 재배치. 재도전/히스토리 로직 자체는 변경 없이 위치만 이동
- 색상 확정(정확한 hex, WCAG 대비 검증, 카테고리 색상 매핑, 시간대별/우주 배경,
  온보딩 행성 미리보기)은 후속 세션 과제로 남김

### 3. 사용자 리포트 버그 3건 수정 (커밋 `420ac0e`)

- **대화 구두점**: 마침표/물음표 없이 문장이 뭉개져 나오는 문제 → 페르소나 규칙에
  "문장이 끝나면 반드시 마침표로 마무리하고 질문에는 물음표를 쓴다"를 명시적으로 추가
  (최초엔 사용자 의도를 반대로 이해해 "구두점을 쓰지 말라"는 규칙을 넣었다가, 사용자가
  코드를 보고 바로 정정해줘서 원래 의도대로 다시 고침)
- **다크/라이트 토글 위치**: 토글이 화면 우상단에 고정된 적이 한 번도 없었던 진짜
  원인을 찾음 — 버튼 자체의 `relative` 클래스가 Tailwind 컴파일 결과에서 `absolute`
  보다 나중에 선언돼 있어, `className`으로 넘긴 `absolute`가 항상 무시되고 있었음
  (HTML class 순서와 무관하게 컴파일된 CSS 선언 순서가 이김). 감싸는 wrapper `div`에
  포지셔닝을 주는 방식으로 해결. 토글 내부 원형 아이콘도 `top-1/2 -translate-y-1/2`로
  캡슐과 수직 중앙 정렬되도록 수정
- **마음 지구 발광 효과**: `.mind-planet__orb::before`(광채)의 `z-index:-1`이 부모가
  별도 스태킹 컨텍스트를 안 만들어서 오브 바로 뒤가 아니라 배경 전체 뒤로 밀려나
  완전히 안 보이던 문제 → `.mind-planet__orb`에 `z-index:0` 추가로 해결. 배경 별도
  무채색(`--color-ink-faint`) 대신 테마별 `--color-highlight`로 바꾸고 크기 확대
  (5/8px 두 종류), 원형(dot)+사각별(sparkle, clip-path) 두 형태를 섞음
